## Language Policy

All Markdown files (.md) in this project must be written in Danish.

This includes, but is not limited to:
- Documentation
- Design descriptions
- Architectural explanations
- Headings and section titles

Source code:
- Code identifiers may be in English
- Comments in source code may be in English

User Interface (UI):
- All user-facing text must be written in Danish when UI is introduced

These rules are mandatory unless explicitly overridden.

## Source Code Naming

All source code identifiers must use English names.

This includes:
- Variable names
- Method and function names
- Class names
- Interface names
- Property names
- File names related to source code

Danish must not be used in source code identifiers.

Comments in source code may be written in English.
